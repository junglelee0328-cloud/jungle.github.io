过去一年，AI 编程工具真正进入了我的日常开发流程。从最早的 ChatGPT 问答，到现在用 Cursor、Claude Code 进行上下文感知的代码生成与重构，我的开发方式发生了明显变化。

## 工具组合

目前我的主力组合是：

| 工具 | 用途 |
|------|------|
| Cursor | 日常编码、重构、单测生成 |
| Claude Code | 复杂需求分析、跨文件改动 |
| MCP | 让模型调用外部工具与知识库 |

## 实际收益

1. ** boilerplate 减少**：CRUD、DTO、配置类可以快速生成初稿
2. **重构更敢做**：AI 能帮我梳理依赖，降低大规模改动的风险
3. **学习加速**：遇到不熟悉的框架时，可以直接问代码库里的具体实现

## 保持警惕

AI 生成的代码仍然需要人把关。我始终坚持：

- 逐行 review，不盲目接受
- 关键路径必须跑测试验证
- 安全、并发、事务边界不能交给 AI 猜

> AI 是杠杆，而不是替身。工程师的判断力才是最重要的资产。
