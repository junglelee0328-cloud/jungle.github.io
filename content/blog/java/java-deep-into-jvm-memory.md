## 内存概述

内存区域是计算机中较为重要的一个部分，它的出现是为了缓冲cpu高速寄存器与低速磁盘之间的速度不匹配问题。

内存区域相对cpu寄存器要大很多，能够存放较多的临时数据。现代的电脑内存大概在8G、16G、32G这样的一个级别，读取速度一般是磁盘的10倍左右。所以，内存条的价格也是非常贵。

一般而言，操作系统占用内存就可能会达到整体内存的一半，所以，合理地管理和利用内存区域是非常重要地。

## Java内存模型

Java内存指的是Java虚拟机管理到的内存，通常是Java虚拟机自身占用内存以及一些虚拟机管理地直接内存。虚拟机直接管理地内存是Java虚拟机在启动时向操作系统申请的一片较大的内存区域，和c这类语言不同的是，c语言是直接管理内存，开发人员甚至可以直接使用指针访问内存区域，这是十分不安全的行为，况且开发者在动态申请内存后很有可能会忘记释放内存，照成内存泄露。

所以，为了防止这一系列的问题，Java设计了一套严密的内存管理机制，由虚拟机直接管理内存，一定程度上减少了上述的一些问题。对象创建和对象销毁这些过程，都由JVM直接管理。

一般而言，如果有特定需求，单台机器上可以部署多台JVM。这样做的好处就是一定程度上提升了Java虚拟机的GC能力，不过缺点就是降低了系统内存的实际利用率。

Java支持多线程，为了保证多线程的可靠性，JVM对线程的内存结构进行了良好的设计。我们知道，多线程的每一个线程，即使使用同样的一份字节码，理论上来讲每一个线程都应有这份代码的独立版本，否则就会出现一系列的脏读乱读现象。所以，对于每一个线程，是需要私有一些数据，做到线程隔离。

那么，那些数据是线程私有的呢？分别是以下几种：

1.  程序计数器：我们知道，class文件的字节码的每一条指令都有一个行号，而程序计数器是一个较小的内存空间，它的作用就是充当行号指示器，线程通过指示器找到需要执行的字节码指令。虚拟机实现多线程是通过线程间的切换、向操作系统申请时间片来实现的，因此为了保证线程的独立性，每个线程必须需要一个私有的程序计数器。
2.  虚拟机栈：虚拟机每执行一个方法，就需要为这个方法创建一个栈帧，这个栈帧保存这个方法体在执行期间的局部变量、动态链接、方法出口等该方法相关的信息。如果在该方法中又调用了另一个函数，那么就会再为被调用的方法创建一个栈帧，等到该调用方法执行完毕，虚拟机栈就会执行出栈操作，栈顶的栈帧就是原来的方法。图示如下：

### 堆内存

堆内存是JVM管理的最大的一块区域，这块区域按分类存储了类对象的实体。通常发生内存溢出等异常有很大部分是该区域的内存泄露。这个区域的具体分区是由具体类别的虚拟机来决定的，其中一些具体的垃圾回收方法也是由具体的虚拟机决定的。

大的分类上来讲，虚拟机的堆内存分为新生代和老年代。新生代通常用来保留一些容易被快速销毁、生命周期短的对象，通常是一些循环、方法局部变量等一些生存周期短但是数量较多的对象。而老年代通常保留一些应用环境变量，应用上下文等生命周期较长但数量较少的对象。

不同虚拟机对堆内存的实现各有不同，现流行的HotSpot虚拟机通常又将新生代分为一个Eden区和两个Sur[vivo](https://www.zhihu.com/search?q=vivo&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22article%22%2C%22sourceId%22%3A%22149948009%22%7D)r区，相当于对新生代对象的年龄进一步细分。

通常，如果发生内存泄露，可以通过添加-Xmx、-Xms来设定堆内存大小。

### 方法区

方法区是线程共享的一块区域，用于存储类型信息，常量、静态变量、即时编译后的代码缓存等。Java虚拟机规范中对方法区的约束较为宽松，通常方法区的变量的生存周期较长，且数量较少，但并非JDK8前的HotSpot定义的“永久代“，因为JDK8之前的HotSpot定义的”永久代“是堆内存的一个分区，但由于该区域经常发生内存溢出，故JDK8后彻底将该区域转化为在本地内存中的元空间。所以，对于不同产品的虚拟机，可以没有”永久代“，”永久代“甚至在不断被淘汰，但是方法区是一定要有的，这是在虚拟机规范中定义好的。

### 直接内存

直接内存不是Java虚拟机规划的内存区域，它是一块外部的内存区域，通常用于优化一些IO输入输出问题，它跳过了Java虚拟机的堆内存，避免了一些数据复制，能够一定程度提高性能。
